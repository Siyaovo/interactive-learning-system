# 交互式学习系统

> 基于苏格拉底式提问法，用一个 markdown 文件让 AI 变成你的私人老师。
> 不用插件，一个文件搞定。

---

## 快速上手

```bash
git clone https://github.com/Siyaovo/interactive-learning-system.git Learning
```

1. 用 Claude Code 打开 `Learning` 文件夹
2. 告诉 Claude：「我想学 XX，帮我开始第一课」
3. Claude 会自动创建课题子文件夹和第一篇课程

**就这样，开始学。**

---

## 工作流

1. AI 生成课题的第一篇文章，末尾附提问环节
2. 用户阅读后，在文章末尾写下回答、问题、感悟
3. 用户说「我学完了」
4. AI 读取用户反馈，分析理解程度和兴趣方向，生成下一篇
5. 重复，直到课题完全掌握

## 文件结构

```
Learning/                  ← 你的学习文件夹（名字随意）
├── README.md              ← 本文件
├── 反脆弱/                 ← 课题文件夹（自动创建）
│   ├── 01-什么是反脆弱.md
│   ├── 02-课题名称.md
│   └── ...
└── 下一个课题/
    └── ...
```

## 文章格式约定

### 文件名

`序号-主题关键词.md`（如 `01-什么是反脆弱.md`）

### 文章结构

```markdown
# 标题

（正文：深度阅读，清晰有深度，像一个懂行的朋友在讲触动他们的点）

---

## 提问环节

> 以下问题帮助你检验自己是否真正理解了这篇内容。
> 不需要写「正确答案」，写你真实的理解就好。

**Q1**: （针对核心概念的理解题）

你的回答：

**Q2**: （联系自身经验的应用题）

你的回答：

**Q3**: （挑战性的延伸思考题）

你的回答：

---

## 自由区

（你的其他问题、感悟、想深入了解的方向，随便写）

```

### 行文风格

- 清晰、有深度，避免空洞的教科书腔调
- 像一个懂行的朋友在讲触动他们的点
- 用真实案例和类比，不用抽象定义堆砌
- 敢下判断，不怕有观点

### AI 回应规则（重要）

**当用户说「学完了」「做完了」或类似表达时，Claude 必须：**

1. 读取用户在该课的所有回答和自由区内容
2. 对每个回答给出回应（对在哪、偏在哪、补充什么、可以怎么修正）
3. 对自由区的问题和感悟也要逐条回应
4. **把这些回应写入该课文件的底部**，格式为 `## Claude 的回应`，写在自由区之后、下一课预告之前
5. 然后再生成下一课

**这一步不能省略。** 用户的回答和 Claude 的回应必须保存在同一个文件里，形成完整的对话记录，方便日后回顾。

### AI 生成下一篇的规则

1. 先读用户在上一篇的所有回答和自由区内容
2. 判断：哪些点已经掌握？哪些点理解偏了？哪些方向用户特别感兴趣？
3. 下一篇基于这些判断调整内容深度和方向
4. 在下一篇开头简要回应用户的回答（对在哪、偏在哪、可以怎么修正）

### 掌握度评估

每篇文章的提问环节结束后，AI 在生成下一篇之前，先输出一个掌握度评估：

```
### 掌握度评估（第 N 轮）

| 核心概念 | 状态 | 说明 |
|---------|------|------|
| {概念1} | ✅ 掌握 / ⚠️ 模糊 / ❌ 未掌握 | {一句话说明} |
| {概念2} | ... | ... |

**整体判断**：继续学习 / 可以毕业
```

**判断标准**：

满足以下全部条件 → 可以毕业：
1. **所有核心概念都是 ✅**（没有 ⚠️ 或 ❌）
2. **用户能用自己的话解释**，而不是复述文章里的原话
3. **用户能举出文章之外的例子**（说明真正内化了，不是记住了）
4. **用户能回答反驳性问题**（「如果有人说 XX 你怎么回应」）

不满足 → 继续生成下一篇，针对 ⚠️ 和 ❌ 的概念加强。

### 过程中的回顾轮

每 3 课自动插入一次回顾（不算新课），AI 必须：

1. 把之前所有 ⚠️ 和 ❌ 的概念重新拿出来，用一个**新的场景**提问（不能重复原来的题目）
2. 挑 1-2 个用户之前回答过的问题，问：「你现在还是这么想吗？有没有想修正的？」
3. 根据用户的新回答，更新掌握度评估表
4. 将回顾内容保存为 `{课题名}/回顾-第N轮.md`

**为什么要有回顾轮：** 第 1 课理解的东西，到第 4 课可能已经被新知识改变了。回顾不是复习，是检验你的理解有没有随着学习而进化。

### 毕业归档

当 AI 判断「可以毕业」时，执行以下流程：

1. **生成毕业笔记**（`{课题名}-毕业笔记.md`），包含：
   - **一句话总结**：这个课题的核心是什么
   - **关键概念清单**：每个概念一句话解释（用你自己的话，不是教科书定义）
   - **我的收获**：从你历次回答中提炼的个人洞察
   - **学习轨迹**：你学习过程中的关键时刻——最初分不清什么、在哪里犯了错、怎么纠正的、最大的发现是什么。半年后再看，这些比概念清单更有用
   - **行动项**：学完之后可以做什么（如果有的话）

2. **归档**：将毕业笔记保存在课题文件夹内，或移动到你的知识库中你认为合适的位置

3. **更新关联**：如果你使用 Obsidian，毕业笔记里可以用 `[[]]` 链接到相关笔记，让知识融入已有的知识网络

### 毕业后 30 天复盘

毕业时，AI 自动在课题文件夹生成一份 `{课题名}-30天复盘卡.md`，内容包括：

1. **行动项检查**：毕业笔记里的行动项，做了吗？效果怎样？
2. **概念抽查**：随机挑 3 个核心概念，用全新的场景提问（不是原来的题目）
3. **应用回顾**：这 30 天里，你有没有在真实场景中用到学过的东西？举一个例子
4. **理解迭代**：重新看一遍毕业笔记里的「我的收获」，有没有想修正或补充的？

用户在 30 天后打开这个文件，写下回答，告诉 AI「复盘完了」。AI 回应后更新毕业笔记。

**为什么要有 30 天复盘：** 刚毕业时你觉得自己全懂了，但真正的理解需要时间和实践来检验。30 天后回来看，才知道哪些真的内化了，哪些只是当时记住了。
